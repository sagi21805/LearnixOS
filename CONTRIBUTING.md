Thank you for your interest in this project! To keep LearnixOS consistent, and pleasant to maintain, please read through these guidelines before opening an Issue or Pull Request.

## Scope of Contributions

Code contributions are welcome. Contributions generally fall into one of these categories:

1. **Backlog items** — Features or fixes already described in an open Issue. Comment on the Issue before starting so the work isn't duplicated.
2. **New ideas** — Features or improvements not yet tracked anywhere. Open an Issue describing the idea *before* writing code, so it can be discussed first.
3. **Fixes** — Bug fixes, typos, and small corrections can go straight to a Pull Request.

**All contributions — code, features, or significant rewrites — must be reviewed and approved by the project owner before merging.** 

## Policy on AI and LLMs

Using AI tools (ChatGPT, Claude, Gemini, Copilot, etc.) to help with your contribution is fine. What matters is the final code. To keep quality high:

1. **You must understand what you submit.** If you can't explain why a change works or defend a design decision in it, don't submit it.
2. **No unreviewed AI output.** Don't paste in raw model output without reading, testing, and editing it first.
3. **Standard tools are always fine** — spellcheckers, linters, formatters, and dictionary tools require no disclosure.

## Submitting Issues

1. **Search first:** Check the "Issues" tab to see if it's already been reported.
2. **Report:** If you find a bug, typo, or missing feature, open an **Issue** or submit a **Pull Request** directly.
3. **Be specific:** Include the current behavior/text, your suggested change, and where it occurs (file, chapter, or page).

## Implementing Features

If you'd like to implement a feature:

- Check the **backlog** (open Issues) for existing feature requests, or propose a new one.
- Comment on the relevant Issue to claim it before you start, so effort isn't duplicated.
- Wait for a sign-off from the project owner before investing significant time — all features need approval before implementation begins, since not every idea fits the project's direction.

Thank you for helping build LearnixOS!
